# Roadmap Atlas

Data ultimo aggiornamento: 2026-05-31

La roadmap descrive direzione, priorità e prossimi passi di Atlas. Le idee non
ancora scelte stanno in `docs/BACKLOG.md`; lo storico esteso di coordinamento
sta in `docs/COORDINATION_HISTORY.md`.

## Ora

- Proseguire l'allineamento dei documenti canonici nelle repo coordinate,
  mantenendo separati roadmap, backlog, storico, decisioni e regole operative
  senza template incollati.
- Usare `docs/STANDARDS.md`, `docs/PUBLISH_DEPLOY_RELEASE.md`,
  `docs/APPLYING_ATLAS.md`, `docs/PROJECTS.md` e `docs/HEALTH.md` come fonti
  operative prima di ogni intervento repo-per-repo.
- Applicare sempre la discovery degli extra repo-specifici prima di
  normalizzazioni, migrazioni o promozione di standard.

## Prossimo

- Aggiornare matrici e registri Atlas quando un ciclo pubblicato cambia stato,
  target, eccezioni o prossime azioni delle repo coordinate.
- Proseguire i punti del piano di coordinamento solo dopo avere chiuso,
  pubblicato e pulito il punto corrente.
- Tenere separati debiti repo e scope Atlas: i debiti delle repo si censiscono o
  si impacchettano, ma non diventano lavoro esecutivo senza target esplicito.

## Più avanti

- Definire un formato standard per report periodico Atlas.
- Valutare automazioni leggere per link check, duplicati documentali e presenza
  dei file canonici.
- Rivalutare periodicamente `docs/STANDARD_CANDIDATES.md` prima di promuovere
  nuovi pattern a standard Atlas.

## Bloccato

- Nessun blocco operativo trasversale aperto.

## Fatto recente

- Creato Atlas come meta-progetto separato con repository GitHub pubblica,
  baseline GitHub leggera e documenti canonici.
- Completato il primo allineamento trasversale di governance, documentazione,
  GitHub, publish/deploy/release e standard candidati.
- Pubblicato l'allineamento delle regole operative generali negli `AGENTS.md`
  delle repo coordinate.
- Pubblicato l'allineamento del catalogo documentale, con indici canonici,
  root/docs chiariti, anti-duplicati e anti-perdita.

## Regole

- La roadmap non è un changelog.
- La roadmap non è un dump di idee.
- Le idee non ancora scelte stanno in `docs/BACKLOG.md`.
- Le decisioni stabili stanno in `docs/decisions/`.
- Lo storico esteso sta in `docs/COORDINATION_HISTORY.md` o nei registri
  canonici collegati.
- Ogni voce attiva deve indicare un prossimo passo operativo reale.
